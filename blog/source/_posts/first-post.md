# WEEK 4 UX

## Modelling Requirement
 ---
### **informal method:** 
1. **user stories**
2. **user cases**
3. **personas**  
    Definition:  
    A Persona Is a fictional character created to convey(传递) a certain set of user properties and behaviours to a technical recipient(接受者)


    


4. **scenarios**
    
    Definition:     
    A Scenario is a detailed description of the user’s interaction with the computer
    

    scenarios can be used to clarify issues and implications of system development when there is no other way to do so.


### **semi-formal method**
1. wireframe（线框)  
线框图很简单，通常是手绘的，是拟议用户界面的草图


2. mock-up and story board
  
  Mock-ups & Storyboards are just extensions of wireframes, but with more flesh on the bones. Again both mock-ups and storyboards may occur as hand drawn sketches, or as computer-generated interface elements. These mostly don’t have the ability to run real code – being presented mainly for their visual look and feel – indeed, both mainly sit between the wireframe and the ‘prototype’. As with wireframes, mockups are mostly a singular interface screen or component (or loosely-related series of screens) which can then be discussed
